# Тест-кейсы для автотестов пользовательских сценариев таск-трекера

# Тест-сюит "Создание задачи"

## Тест-кейс 1: Создание задачи

### Цель:
Проверить, что пользователь может успешно создать задачу через форму при нажатии на кнопку "СОЗДАТЬ ЗАДАЧУ".

### Предусловия:
1. На компьютере открыт браузер Google Chrome версии не ниже 127 в режиме инкогнито.
2. Открыта страница списка задач по адресу: https://avito-tech-internship-psi.vercel.app/issues

### Шаги:
1. Нажать кнопку "**"СОЗДАТЬ ЗАДАЧУ"**".
2. В поле **"Название"** ввести текст `бэм бэм бэм`.
3. В поле **"Описание"** ввести текст `быстрее`.
4. В поле **"Проект"** выбрать `Миграция на новую БД`.
5. В поле **"Приоритет"** выбрать `High`.
5. В поле **"Исполнитель"** выбрать `Максим Орлов`.
4. Нажать кнопку **"СОЗДАТЬ"**.
5. Дождаться перехода на карточку созданной задачи.

### Ожидаемый результат:
- Открывается страница карточки задачи.  
- Заголовок задачи отображается как `бэм бэм бэм`.  
- Статус задачи установлен в `Backlog`.  
- Созданная задача отображается в общем списке задач.

## Тест-кейс 2: Открытие карточки задачи

### Цель:
Проверить, что пользователь может открыть карточку существующей задачи из списка.

### Предусловия:
1. В системе существует задача с названием `бэм бэм бэм`.
2. Открыта страница списка задач `/issues`.

### Шаги:
1. Найти в списке задачу с названием `бэм бэм бэм`.
2. Нажать на название задачи для перехода на её карточку.

### Ожидаемый результат:
- Открывается карточка задачи.  
- Отображаются поля: **Название**, **Описание**, **Проект**, **Приоритет**, **Статус**, **Исполнитель**.  
- Название соответствует выбранной задаче.

## Тест-кейс 3: Поиск задачи

### Цель:
Проверить, что поиск задач по названию работает корректно.

### Тест-кейс 3.1: Поиск существующей задачи

#### Предусловия:
1. В системе существует задача с названием `бэм бэм бэм`.
2. Открыта страница `/issues`.

#### Шаги:
1. Ввести строку `бэм бэм бэм` в поле поиска.
2. Нажать Enter.

#### Ожидаемый результат:
- В результатах отображается задача `бэм бэм бэм`.  
- Другие задачи в списке отсутствуют.

### Тест-кейс 3.2: Поиск несуществующей задачи

#### Предусловия:
1. Открыта страница `/issues`.

#### Шаги:
1. Ввести строку `qwerty123` в поле поиска.
2. Нажать Enter.

#### Ожидаемый результат:
- Список задач пуст.  
- На странице отображается сообщение об отсутствии результатов (например: *Задачи не найдены.*).

## Тест-кейс 4: Переход на доску проекта

### Цель:
Проверить, что пользователь может просмотреть список проектов и открыть доску выбранного проекта.

### Предусловия:
1. Открыта страница `/issues`.

### Шаги:
1. В хедере страницы нажать кнопку **"ПРОЕКТЫ"**.
2. Дождаться перехода на страницу `/boards`.
3. Найти проект с названием **"Миграция на новую БД"**.
4. Нажать кнопку **"Перейти к доске"**.
5. Убедиться, что происходит переход на страницу доски проекта (URL вида `/boards/<id>`).

### Ожидаемый результат:
- Открыта доска выбранного проекта.  
- На странице отображаются колонки канбан-доски.  
- Заголовок доски соответствует выбранному проекту.
